# Brutal Audit Skill

Жёсткий аудит структуры проекта. Представь внешнего аудитора который первый раз смотрит на код и доёбывается до всего.

## Когда использовать
- Как часть maintenance workflow (nightly)
- После большого рефакторинга
- Когда "что-то не так, но непонятно что"
- Перед code review

## Что проверяет

### CRITICAL (score -20)
- Мёртвые skills (без run.sh и skill.json)
- Неполные workflows (без workflow.json)
- Сломанная архитектура (отсутствующие ключевые директории)

### HIGH (score -10)
- Неизвестные директории с файлами
- Skills без SKILL.md
- Deprecated директории которые ещё существуют
- Свой package.json в scripts/
- Несогласованные конфиги

### MEDIUM (score -5)
- Лишние файлы в корне
- LinkedIn и другие случайные скрипты
- Раздутые директории (>100 файлов)

### LOW (score -2)
- Пустые директории
- Неиспользуемые скрипты
- Нарушения именования (не kebab-case)

## Выход

```json
{
  "issues": [
    {
      "severity": "CRITICAL",
      "category": "dead_skill",
      "message": "Skill 'nightly-consilium' без run.sh и skill.json",
      "fix": "Добавить исполняемый файл или удалить skill"
    }
  ],
  "summary": {
    "total": 15,
    "critical": 2,
    "high": 5,
    "medium": 5,
    "low": 3
  },
  "score": 45
}
```

## Философия

> "Если ты не понимаешь зачем это здесь - это мусор"

- Каждый файл должен иметь чёткое назначение
- Каждая директория должна быть в архитектуре
- Мёртвый код = технический долг
- Недокументированный код = невидимый код
